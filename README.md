## 修改用户密码

容器运行期间，在宿主机执行：

```bash
docker exec -it yokan-backend ./change_password
```

交互式流程：列出所有用户 → 输入用户名或序号 → 输入新密码（不回显）→ 确认密码。

密码要求：长度 ≥ 8 位，包含大写/小写/数字/特殊字符中至少 3 种。

## 功能模块

### 自定义工具导入

在「工具管理」页面，将任意本地命令行工具接入平台统一调度。

**接入模式：**

| 模式       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| `external` | 直接外部导入linux（amd64）/windwos、java、python（必须要带有requirements.txt文件） |

<img width="1313" height="744" alt="image" src="https://github.com/user-attachments/assets/5ec746f6-f7ef-4df2-8b4e-ed969f0e2397" />


**外部模板（一键导入）：**

| 工具 | 命令格式                              |
| ---- | ------------------------------------- |
| Nmap | `nmap -T4 {参数} -p {参数} --open` |

占位符 `{参数名}` 由平台在执行时自动填充，支持高级模式配置参数映射和输出解析器。

---

### 自定义工作流（攻击链）

在「信息收集 → 攻击链管理」中，将多个工具/模块串联成自动化流程。

- 可视化节点编排，拖拽连线
- 树状分支流程，上游输出自动作为下游输入
- 支持 9 种模块类型：子域名 → 存活检测 → 端口扫描 → 指纹识别 → 漏洞扫描 → ...
- 实时执行进度，结果跨节点聚合

内置示例：「智能目标识别攻击链」，公司名 → 子域名 → 存活 → 端口 → 漏洞，全自动。

### 漏洞扫描

| 功能     | 说明                                                   |
| -------- | ------------------------------------------------------ |
| POC 管理 | 自定义 POC 导入、去重、缓存                            |
| POC 扫描 | 批量目标 × 批量 POC，任务队列管理，采用nuclei形势的poc |
| 漏洞管理 | 扫描结果归档，按项目/严重性筛选                        |

---

### 文件泄露检测

| 功能     | 说明                                                    |
| -------- | ------------------------------------------------------- |
| 路径扫描 | 13 类字典分类扫描（配置/备份/管理后台/日志/数据库等）   |
| AI Fuzz  | 基于已发现路由做命名风格克隆，智能推断高概率路径        |
| Replay   | HTTP 原始报文重放，`§payload§` 标记注入点，实时进度显示 |
| 字典管理 | 内置字典支持自定义追加                                  |
| 路由发现 | 自动提取 JS/HTML 中的隐藏路由                           |

AI Fuzz 工作原理：识别站点命名风格（驼峰/下划线/短横线）→ 业务模块横向扩展（user→role/permission）→ CRUD 变体推断（list→add/edit/delete/export）。
